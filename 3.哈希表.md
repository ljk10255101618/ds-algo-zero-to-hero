## 二、哈希表的基本原理

### 2.1 什么是哈希表？

**专业解释**：哈希表（Hash Table）是一种根据关键码直接访问值的数据结构，通过哈希函数将键映射到表中的位置来访问记录。

```
哈希表的工作原理：

    键(key)              哈希函数                  数组(哈希表)
  ┌────────┐          ┌──────────┐           ┌─────────────────┐
  │  "apple" │ ──────→ │ hash(key) │ ──────→  │ 0 │ 1 │ 2 │ 3 │ 4 │ ...
  └────────┘          └──────────┘           └─────────────────┘
                                                   ↑
                                              hash("apple") = 2
                                              直接访问 arr[2]！
```

### 2.2 哈希函数

**什么是哈希函数？**
把任意长度的输入，通过算法变成固定长度的输出。这个输出就是"哈希值"。

```
哈希函数示例：
┌──────────────────┬────────────────┐
│ 输入             │ 哈希值         │
├──────────────────┼────────────────┤
│ "apple"          │ 748921234      │
│ "banana"         │ 123456789      │
│ 12345            │ 987654321      │
└──────────────────┴────────────────┘

特点：
- 相同输入 → 相同输出 ✅
- 不同输入 → 不同输出（理想情况）
- 单向计算，无法反推
```

### 2.3 简单的哈希函数实现

```python
def simple_hash(key, table_size):
    """
    简单的哈希函数：求余法
    hash(key) = key mod table_size
    """
    return key % table_size

# 测试
table_size = 7
for key in [15, 22, 8, 36, 29]:
    print(f"hash({key}) = {key} mod {table_size} = {simple_hash(key, table_size)}")

# 输出：
# hash(15) = 15 mod 7 = 1
# hash(22) = 22 mod 7 = 1  ← 冲突！
# hash(8) = 8 mod 7 = 1   ← 又冲突！
# hash(36) = 36 mod 7 = 1 ← 还是冲突！
# hash(29) = 29 mod 7 = 1 ← 太多冲突了！
```

### 2.4 更好的哈希函数

```python
def better_hash(key, table_size):
    """
    改进的哈希函数：乘法+求余
    hash(key) = (key * 黄金分割数) mod table_size
    """
    if isinstance(key, str):
        # 字符串：用 ASCII 码
        key = sum(ord(c) for c in key)
    return int(key * 0.618) % table_size

# 测试
table_size = 7
keys = ["apple", "banana", "cherry", "date", "elderberry"]
for key in keys:
    print(f"hash('{key}') = {better_hash(key, table_size)}")
```

## 三、哈希冲突

### 3.1 什么是哈希冲突？

**类比**：两个同学学号相同，都坐同一把椅子。

```
插入 "apple" → hash = 2 → arr[2] = "apple"
插入 "banana" → hash = 2 → arr[2] 已经被占了！冲突！
```

### 3.2 解决哈希冲突的方法

```
方法1：开放寻址法（Open Addressing）
     冲突了？找下一个空位！

     ┌───┬───┬───┬───┬───┐
     │ A │ B │ C │   │   │  ← C 占用了位置 2
     └───┴───┴───┴───┴───┘
       0   1   2   3   4
     
     插入 D, hash(D)=2，位置被占
     → 检查 3，空的，放这里！

方法2：链地址法（Separate Chaining）
     冲突了？用链表串起来！

     ┌───┐
     │ 0 │ → NULL
     ├───┤
     │ 1 │ → [B] → [F] → NULL
     ├───┤
     │ 2 │ → [A] → [C] → [D] → NULL
     ├───┤
     │ 3 │ → [E] → NULL
     └───┘
```

### 3.3 开放寻址法实现

```python
class HashTableOpenAddressing:
    """开放寻址法实现的哈希表"""
    
    def __init__(self, size=10):
        self.size = size
        self.keys = [None] * size
        self.values = [None] * size
    
    def _hash(self, key):
        """哈希函数"""
        if isinstance(key, str):
            key = sum(ord(c) for c in key)
        return key % self.size
    
    def _probe(self, index, i):
        """探测：线性探测法"""
        return (index + i) % self.size
    
    def put(self, key, value):
        """插入"""
        index = self._hash(key)
        i = 0
        while i < self.size:
            new_index = self._probe(index, i)
            if self.keys[new_index] is None or self.keys[new_index] == key:
                self.keys[new_index] = key
                self.values[new_index] = value
                return
            i += 1
        raise OverflowError("Hash table is full")
    
    def get(self, key):
        """查找"""
        index = self._hash(key)
        i = 0
        while i < self.size:
            new_index = self._probe(index, i)
            if self.keys[new_index] is None:
                return None  # 不存在
            if self.keys[new_index] == key:
                return self.values[new_index]
            i += 1
        return None
    
    def delete(self, key):
        """删除（标记墓碑）"""
        index = self._hash(key)
        i = 0
        while i < self.size:
            new_index = self._probe(index, i)
            if self.keys[new_index] is None:
                return False
            if self.keys[new_index] == key:
                self.keys[new_index] = None  # 墓碑
                self.values[new_index] = None
                return True
            i += 1
        return False


# 测试
ht = HashTableOpenAddressing(5)
ht.put("apple", 1)
ht.put("banana", 2)
ht.put("cherry", 3)
print(ht.get("apple"))   # 1
print(ht.get("banana"))  # 2
print(ht.get("cherry"))  # 3
```

### 3.4 链地址法实现

```python
class HashTableChaining:
    """链地址法实现的哈希表"""
    
    def __init__(self, size=10):
        self.size = size
        self.buckets = [[] for _ in range(size)]  # 每个桶是链表
    
    def _hash(self, key):
        """哈希函数"""
        if isinstance(key, str):
            key = sum(ord(c) for c in key)
        return key % self.size
    
    def put(self, key, value):
        """插入或更新"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        # 查找是否已存在
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 更新
                return
        
        # 不存在，添加到链表末尾
        bucket.append((key, value))
    
    def get(self, key):
        """查找"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        for k, v in bucket:
            if k == key:
                return v
        return None
    
    def delete(self, key):
        """删除"""
        index = self._hash(key)
        bucket = self.buckets[index]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                return True
        return False
    
    def __repr__(self):
        return repr(self.buckets)


# 测试
ht = HashTableChaining(5)
ht.put("apple", 1)
ht.put("banana", 2)
ht.put("cherry", 3)
ht.put("date", 4)
print(ht)
# [[], [(2, 2), (7, 4)], [], [(0, 1), (5, 3)], []]
print(ht.get("apple"))   # 1
print(ht.get("banana"))  # 2
ht.delete("banana")
print(ht.get("banana"))  # None
```

## 四、Python 内置哈希表：字典

### 4.1 字典是哈希表

```python
# Python 的 dict 就是哈希表！
d = {}

# 插入/更新 - O(1)
d["apple"] = 1
d["banana"] = 2
d["cherry"] = 3

# 查找 - O(1)
print(d["apple"])        # 1
print(d.get("grape"))    # None
print(d.get("grape", 0)) # 0（默认值）

# 删除 - O(1)
del d["banana"]

# 遍历
for key, value in d.items():
    print(f"{key}: {value}")

# 检查是否存在
print("apple" in d)      # True
```

### 4.2 字典的底层实现

```
Python dict 底层原理：

1. 哈希函数：Python 自己的哈希算法（可移植、防御DoS）

2. 冲突解决：开放寻址法 + 随机探测

3. 动态扩容：
   - 当填充率 > 2/3 时扩容
   - 容量通常是 2^n（方便位运算）
   - 扩容后重新哈希所有元素

4. 特殊机制：
   - 墓碑标记（删除优化）
   - 延迟扩容
```

### 4.3 字典的性能

```python
import time

# 创建大字典测试性能
d = {}

# 插入 100 万个元素
start = time.time()
for i in range(1_000_000):
    d[i] = i
print(f"插入100万: {time.time() - start:.3f}s")

# 查找
start = time.time()
for i in range(100_000):
    _ = d[500000]
print(f"查找10万次: {time.time() - start:.3f}s")

# 结果：插入和查找都非常快，接近 O(1)
```

## 五、集合（Set）

### 5.1 集合也是哈希表

```python
# Python 的 set 就是哈希表（只存键）
s = set()

# 添加 - O(1)
s.add(1)
s.add(2)
s.add(3)

# 查找 - O(1)
print(2 in s)   # True
print(5 in s)   # False

# 删除 - O(1)
s.remove(2)

# 集合运算
s1 = {1, 2, 3, 4}
s2 = {3, 4, 5, 6}

print(s1 & s2)  # {3, 4}  交集
print(s1 | s2)  # {1, 2, 3, 4, 5, 6}  并集
print(s1 - s2)  # {1, 2}  差集
print(s1 ^ s2)  # {1, 2, 5, 6}  对称差集
```

### 5.2 集合的经典应用

```python
# 1. 去重
items = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4]
unique = list(set(items))
print(unique)  # [1, 2, 3, 4]

# 2. 判断是否重复
def has_duplicate(nums):
    return len(nums) != len(set(nums))

print(has_duplicate([1, 2, 3, 4]))  # False
print(has_duplicate([1, 2, 2, 3]))  # True

# 3. 两数和问题（高效版本）
def two_sum(nums, target):
    """
    找出数组中两数之和等于 target 的两个数的索引
    """
    seen = set()
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return True
        seen.add(num)
    return False

print(two_sum([2, 7, 11, 15], 9))  # True (2 + 7 = 9)
```

## 六、哈希表的优缺点

### 6.1 优点

```
✅ 查找速度快：平均 O(1)
✅ 插入删除快：平均 O(1)
✅ 键值对映射自然
✅ 实现简单（Python 一行代码）
```

### 6.2 缺点

```
❌ 哈希冲突影响性能（最坏 O(n)）
❌ 无序（除非用有序字典）
❌ 迭代可能不是按插入顺序
❌ 哈希函数计算有开销
❌ 占用更多内存（有空槽）
```

### 6.3 使用场景

```
适合使用哈希表的场景 ✅：
  - 需要快速查找（如缓存、计数）
  - 需要键值对映射
  - 需要去重
  - 需要 O(1) 的存在性判断

不适合使用哈希表的场景 ❌：
  - 需要有序遍历
  - 数据量很小（直接数组更快）
  - 内存紧张
  - 需要范围查找（如找 1-100 之间的数）
```

## 七、知识总结

### 7.1 一句话总结

```
哈希表 = 数组 + 哈希函数
       = 键 → 数组索引 → 直接访问值
       = O(1) 查找的秘密
```

### 7.2 复杂度速查

| 操作 | 平均 | 最坏 |
|------|------|------|
| 查找 | O(1) | O(n) |
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |

### 7.3 常见哈希函数

```
1. 除法哈希：h(k) = k mod m
2. 乘法哈希：h(k) = floor(m * (k * A mod 1))
3. 字符串哈希：h(s) = Σ(ord(s[i]) * p^i) mod m
```

### 7.4 解决冲突的方法

```
1. 开放寻址法：
   - 线性探测：h(k,i) = (h'(k) + i) mod m
   - 二次探测：h(k,i) = (h'(k) + c1*i + c2*i²) mod m
   - 双重哈希：h(k,i) = (h1(k) + i*h2(k)) mod m

2. 链地址法：
   - 每个槽位放一个链表

3. 再哈希法：
   - 用多个哈希函数依次尝试
```

## 八、自我检验

### 思考题

1. **基础题**：为什么好的哈希函数很重要？
2. **进阶题**：哈希表和数组相比，有什么优势？什么情况下数组更好？
3. **应用题**：如何用哈希表实现 LRU 缓存？
4. **思考题**：Python 的 dict 是线程安全的吗？为什么？

### 参考答案

1. **好的哈希函数**：
   - 减少冲突，让数据均匀分布
   - 计算简单，开销小
   - 一致性：相同输入产生相同输出

2. **哈希表 vs 数组**：
   - 哈希表：O(1) 查找，但无序，内存开销大
   - 数组：有序，可二分查找 O(log n)，但插入删除 O(n)

3. **LRU 缓存**：
   - 用哈希表 + 双向链表
   - 哈希表：O(1) 查找 key
   - 双向链表：O(1) 维护访问顺序

4. **线程安全**：
   - Python 3.7+ 的 dict 不是线程安全的
   - 多线程同时写入可能出问题
   - 读取可以安全共享
