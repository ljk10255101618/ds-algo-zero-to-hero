## 一、为什么要排序？

### 1.1 生活中的排序

```
成绩排名：按分数从高到低
字典：按字母顺序排列
书架：按类别/作者排列

排序后：
- 更容易找到目标
- 便于统计分析
- 是很多算法的前置步骤
```

### 1.2 排序的分类

```
比较排序：通过比较元素大小排序
  - 冒泡排序 O(n²)
  - 选择排序 O(n²)
  - 插入排序 O(n²)
  - 归并排序 O(n log n)
  - 快速排序 O(n log n) 平均
  - 堆排序 O(n log n)

非比较排序：不通过比较排序
  - 计数排序 O(n + k)
  - 基数排序 O(nk)
  - 桶排序 O(n)
```

## 二、基础排序算法

### 2.1 冒泡排序（Bubble Sort）

**思想**：像水底的气泡，大的往上冒（每趟把最大的冒到最后）

```
第一趟：
[5, 3, 8, 4, 2]
 ↓        ↓     5>3, 交换
[3, 5, 8, 4, 2]
    ↓     ↓    5>8, 不交换
[3, 5, 8, 4, 2]
       ↓ ↓    8>4, 交换
[3, 5, 4, 8, 2]
          ↓↑  8>2, 交换
[3, 5, 4, 2, 8]  ← 8 到顶了

第二趟：[3, 5, 4, 2, 8] → [3, 4, 2, 5, 8]
第三趟：[3, 4, 2, 5, 8] → [3, 2, 4, 5, 8]
第四趟：[3, 2, 4, 5, 8] → [2, 3, 4, 5, 8]
完成！
```

```python
def bubble_sort(arr):
    """冒泡排序"""
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:  # 已经有序，提前退出
            break
    return arr


def bubble_sort_optimized(arr):
    """优化版：记录最后交换位置"""
    n = len(arr)
    last_swap = n - 1
    while last_swap > 0:
        new_last_swap = 0
        for j in range(last_swap):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                new_last_swap = j
        last_swap = new_last_swap
    return arr


# 测试
arr = [64, 34, 25, 12, 22, 11, 90]
print("原数组:", arr)
print("排序后:", bubble_sort(arr.copy()))
# [11, 12, 22, 25, 34, 64, 90]
```

**时间复杂度**：O(n²)，优化后最好 O(n)

### 2.2 选择排序（Selection Sort）

**思想**：每次选择最小的，放到前面

```
初始：[64, 25, 12, 22, 11]
      ↑ 最小是 11，放到位置 0
[11, 25, 12, 22, 64]
         ↑ 最小是 12，放到位置 1
[11, 12, 25, 22, 64]
            ↑ 最小是 22，放到位置 2
[11, 12, 22, 25, 64]
               ↑ 最小是 25，放到位置 3
[11, 12, 22, 25, 64]
完成！
```

```python
def selection_sort(arr):
    """选择排序"""
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr


# 测试
arr = [64, 25, 12, 22, 11]
print(selection_sort(arr.copy()))
# [11, 12, 22, 25, 64]
```

**时间复杂度**：O(n²)，但交换次数少（最多 n 次）

### 2.3 插入排序（Insertion Sort）

**思想**：像打牌时整理手牌，把每张牌插到合适位置

```
手牌：[5, 3, 8, 4, 2]
      ↑
摸到 3，插到 [5] 前
[3, 5, 8, 4, 2]
         ↑
摸到 8，位置不变
[3, 5, 8, 4, 2]
      ↑
摸到 4，插到 [3, 5] 中
[3, 4, 5, 8, 2]
   ↑
摸到 2，插到最前面
[2, 3, 4, 5, 8]
完成！
```

```python
def insertion_sort(arr):
    """插入排序"""
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr


def insertion_sort_binary(arr):
    """二分插入排序（减少比较次数）"""
    for i in range(1, len(arr)):
        key = arr[i]
        # 二分查找插入位置
        left, right = 0, i - 1
        while left <= right:
            mid = (left + right) // 2
            if arr[mid] > key:
                right = mid - 1
            else:
                left = mid + 1
        # 移动元素
        for j in range(i - 1, left - 1, -1):
            arr[j + 1] = arr[j]
        arr[left] = key
    return arr


# 测试
arr = [64, 25, 12, 22, 11]
print(insertion_sort(arr.copy()))
# [11, 12, 22, 25, 64]
```

**时间复杂度**：O(n²)，但对基本有序的数组接近 O(n)

## 三、高级排序算法

### 3.1 归并排序（Merge Sort）

**思想**：分治法，先分后合，分到单个再合并

```
分：[38, 27, 43, 3, 9, 82, 10]
    ↓ ↓ ↓ ↓ ↓ ↓ ↓
分：[38, 27, 43, 3] [9, 82, 10]
    ↓ ↓ ↓ ↓   ↓ ↓ ↓
分：[38, 27] [43, 3] [9, 82] [10]
    ↓ ↓   ↓ ↓   ↓ ↓   ↓
分：[38] [27] [43] [3] [9] [82] [10]
    ↓ ↓   ↓ ↓   ↓ ↓   ↓
合：[27, 38] [3, 43] [9, 82] [10]
    ↓ ↓     ↓ ↓     ↓ ↓
合：[3, 27, 38, 43] [9, 10, 82]
    ↓ ↓     ↓ ↓     ↓ ↓
合：[3, 9, 10, 27, 38, 43, 82]
完成！
```

```python
def merge_sort(arr):
    """归并排序"""
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)


def merge(left, right):
    """合并两个有序数组"""
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result


def merge_sort_inplace(arr):
    """原地归并排序"""
    def merge_range(arr, left, mid, right):
        i, j = left, mid + 1
        while i <= j <= right or (i <= mid and j <= right):
            if j > right or (i <= mid and arr[i] <= arr[j]):
                i += 1
            else:
                arr.insert(i, arr.pop(j))
                i += 1
                mid += 1
    
    def sort(arr, left, right):
        if left >= right:
            return
        mid = (left + right) // 2
        sort(arr, left, mid)
        sort(arr, mid + 1, right)
        merge_range(arr, left, mid, right)
    
    sort(arr, 0, len(arr) - 1)
    return arr


# 测试
arr = [38, 27, 43, 3, 9, 82, 10]
print(merge_sort(arr))
# [3, 9, 10, 27, 38, 43, 82]
```

**时间复杂度**：O(n log n)，稳定排序

### 3.2 快速排序（Quick Sort）

**思想**：选一个基准（pivot），把小于基准放左边，大于放右边，递归

```
选择基准：5（第一个元素）
分区：[3, 2, 1 | 5 | 6, 8, 7, 9]
      小于5   基准 大于5

递归排序左边：[3, 2, 1]
选择基准：3
分区：[1, 2 | 3]
      小于3  基准

递归排序右边：[6, 8, 7, 9]
选择基准：6
分区：[6 | 8, 7, 9]
选择基准：8
分区：[7 | 8 | 9]

合并：[1, 2, 3, 5, 6, 7, 8, 9]
完成！
```

```python
def quick_sort(arr):
    """快速排序"""
    if len(arr) <= 1:
        return arr
    
    pivot = arr[len(arr) // 2]  # 选择中间元素作为基准
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    return quick_sort(left) + middle + quick_sort(right)


def quick_sort_inplace(arr):
    """原地快速排序"""
    def partition(arr, low, high):
        pivot = arr[high]
        i = low - 1
        for j in range(low, high):
            if arr[j] <= pivot:
                i += 1
                arr[i], arr[j] = arr[j], arr[i]
        arr[i + 1], arr[high] = arr[high], arr[i + 1]
        return i + 1
    
    def sort(arr, low, high):
        if low < high:
            pi = partition(arr, low, high)
            sort(arr, low, pi - 1)
            sort(arr, pi + 1, high)
    
    sort(arr, 0, len(arr) - 1)
    return arr


def quick_sort_random(arr):
    """随机化快速排序（避免最坏情况）"""
    import random
    
    def partition(arr, low, high):
        rand_idx = random.randint(low, high)
        arr[rand_idx], arr[high] = arr[high], arr[rand_idx]
        pivot = arr[high]
        i = low - 1
        for j in range(low, high):
            if arr[j] <= pivot:
                i += 1
                arr[i], arr[j] = arr[j], arr[i]
        arr[i + 1], arr[high] = arr[high], arr[i + 1]
        return i + 1
    
    def sort(arr, low, high):
        if low < high:
            pi = partition(arr, low, high)
            sort(arr, low, pi - 1)
            sort(arr, pi + 1, high)
    
    sort(arr, 0, len(arr) - 1)
    return arr


# 测试
arr = [38, 27, 43, 3, 9, 82, 10]
print(quick_sort(arr.copy()))
# [3, 9, 10, 27, 38, 43, 82]
```

**时间复杂度**：
- 平均：O(n log n)
- 最坏（有序数组）：O(n²)
- 优化：随机化、三数取中

### 3.3 堆排序（Heap Sort）

**思想**：利用堆的特性排序，先建最大堆，再逐个取出

```
建堆：[4, 10, 3, 5, 1]
      4
     / \
    10  3
   / \
  5   1

调整后（最大堆）：
       10
      / \
     5   3
    / \
   4   1

排序：取出 10，放到最后
       1
      / \
     5   3
    /
   4
调整...得到：
       5
      / \
     4   3
    /
   1
取出 5...重复
```

```python
def heap_sort(arr):
    """堆排序"""
    n = len(arr)
    
    def heapify(arr, n, i):
        """维护最大堆"""
        largest = i
        left = 2 * i + 1
        right = 2 * i + 2
        
        if left < n and arr[left] > arr[largest]:
            largest = left
        if right < n and arr[right] > arr[largest]:
            largest = right
        
        if largest != i:
            arr[i], arr[largest] = arr[largest], arr[i]
            heapify(arr, n, largest)
    
    # 建最大堆
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    
    # 逐个取出堆顶
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 移到末尾
        heapify(arr, i, 0)  # 维护剩余堆
    
    return arr


# 使用 heapq
import heapq

def heap_sort_heapq(arr):
    """用 heapq 实现堆排序"""
    heap = arr.copy()
    heapq.heapify(heap)
    return [heapq.heappop(heap) for _ in range(len(heap))]


# 测试
arr = [4, 10, 3, 5, 1]
print(heap_sort(arr.copy()))
# [1, 3, 4, 5, 10]
```

**时间复杂度**：O(n log n)，不稳定排序

## 四、非比较排序

### 4.1 计数排序（Counting Sort）

**思想**：统计每个数出现次数，再按次数输出

```
输入：[4, 2, 2, 8, 3, 3, 1]
统计：
  0: 0, 1: 1, 2: 2, 3: 2, 4: 1, 5: 0, 6: 0, 7: 0, 8: 1

前缀和：
  0: 0, 1: 1, 2: 3, 3: 5, 4: 6, 5: 6, 6: 6, 7: 6, 8: 7

输出：[1, 2, 2, 3, 3, 4, 8]
```

```python
def counting_sort(arr):
    """计数排序（假设数据范围 0-100）"""
    if not arr:
        return []
    
    max_val = max(arr)
    count = [0] * (max_val + 1)
    
    # 统计
    for num in arr:
        count[num] += 1
    
    # 前缀和
    for i in range(1, len(count)):
        count[i] += count[i - 1]
    
    # 输出
    result = [0] * len(arr)
    for num in arr:
        count[num] -= 1
        result[count[num]] = num
    
    return result


def counting_sort_range(arr, min_val, max_val):
    """计数排序（指定范围）"""
    if not arr:
        return []
    
    range_size = max_val - min_val + 1
    count = [0] * range_size
    result = [0] * len(arr)
    
    for num in arr:
        count[num - min_val] += 1
    
    for i in range(1, range_size):
        count[i] += count[i - 1]
    
    for num in arr:
        count[num - min_val] -= 1
        result[count[num - min_val]] = num
    
    return result


# 测试
arr = [4, 2, 2, 8, 3, 3, 1]
print(counting_sort(arr))
# [1, 2, 2, 3, 3, 4, 8]
```

**时间复杂度**：O(n + k)，k 是数据范围

### 4.2 桶排序（Bucket Sort）

**思想**：把数据分到有限个桶里，每个桶内排序，再合并

```
输入：[0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12]

分桶（每个桶容量 0.25）：
桶0 [0.00-0.25): 0.17, 0.21, 0.12
桶1 [0.25-0.50): 0.26, 0.39
桶2 [0.50-0.75): 0.72, 0.78
桶3 [0.75-1.00): 0.94

桶内排序：[0.12, 0.17, 0.21], [0.26, 0.39], [0.72, 0.78], [0.94]

合并：[0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]
```

```python
def bucket_sort(arr, num_buckets=5):
    """桶排序"""
    if not arr:
        return []
    
    min_val, max_val = min(arr), max(arr)
    bucket_size = (max_val - min_val) / num_buckets + 1
    buckets = [[] for _ in range(num_buckets)]
    
    # 分桶
    for num in arr:
        idx = int((num - min_val) / bucket_size)
        buckets[idx].append(num)
    
    # 桶内排序
    for bucket in buckets:
        bucket.sort()
    
    # 合并
    result = []
    for bucket in buckets:
        result.extend(bucket)
    
    return result


# 测试
arr = [0.78, 0.17, 0.39, 0.26, 0.72, 0.94, 0.21, 0.12]
print(bucket_sort(arr))
# [0.12, 0.17, 0.21, 0.26, 0.39, 0.72, 0.78, 0.94]
```

## 五、排序算法对比

### 5.1 复杂度总结

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |
| 计数排序 | O(n + k) | O(n + k) | O(n + k) | O(k) | ✅ |
| 桶排序 | O(n + k) | O(n + k) | O(n²) | O(n + k) | ✅ |

### 5.2 选择指南

```
数据规模小（n < 50）：
  → 插入排序，代码简单，小规模性能好

基本有序：
  → 插入排序，接近 O(n)

数据规模大但内存充足：
  → 归并排序，稳定且保证 O(n log n)

通用场景（平均性能）：
  → 快速排序，实际中很快

追求稳定排序：
  → 归并排序

整数/小范围数据：
  → 计数排序、桶排序
```

## 六、知识总结

### 6.1 一句话总结

```
冒泡：泡泡往上冒，大的先到顶
选择：每轮选最小，放到最前
插入：像打牌，逐个插入有序区
归并：先分后合，小有序合大有序
快排：选基准，左小右大
堆排：利用堆，O(n log n) 但不稳定
计数：统计个数，按个数输出
桶排：分桶排序再合并
```

### 6.2 稳定 vs 不稳定

```
稳定排序：相等的元素相对位置不变
  冒泡、插入、归并、计数、桶排序 ✅

不稳定排序：相等的元素相对位置可能变
  选择、快排、堆排序 ❌

什么时候需要稳定？
  多关键字排序：先按学号排，再按成绩排
  如果不稳定，成绩排序会打乱学号顺序
```

## 七、自我检验

### 思考题

1. **基础题**：什么是稳定排序？为什么稳定排序重要？
2. **进阶题**：为什么快速排序实际应用中比归并排序快？
3. **应用题**：如何对 100 万个手机号排序？
4. **思考题**：为什么 Java 的 Arrays.sort() 对小数组用插入排序？

### 参考答案

1. **稳定排序**：相等元素的相对位置不变。重要场景：多关键字排序。

2. **快速排序更快**：
   - 局部性更好，CPU 缓存命中率高
   - 系数小（比较和交换次数少）
   - 原地排序，空间开销小

3. **手机号排序**：
   - 手机号是 11 位整数，可以用字符串或整数表示
   - 建议用基数排序（O(k*n)，k=11）
   - 或者桶排序（分 10 个桶，按数字逐位排序）

4. **小数组用插入排序**：
   - 小数组（< 47 个元素）插入排序更快
   - 归并排序的递归开销不值得
   - 插入排序对小数组接近 O(n)
