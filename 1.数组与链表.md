## 二、数组 —— 连续的内存空间

### 2.1 什么是数组？

**类比**：想象一排连续的储物柜，每个柜子都有固定的编号（0, 1, 2, 3...）。

```
数组示意图：
┌───┬───┬───┬───┬───┬───┐
│ 1 │ 3 │ 5 │ 7 │ 9 │ 2 │  ← 一排连续的内存格子
└───┴───┴───┴───┴───┴───┘
  0   1   2   3   4   5     ← 下标（索引）
```

**专业解释**：数组是一种线性数据结构，用一段连续的内存空间存储相同类型的元素。

### 2.2 数组的特点

| 特点 | 说明 |
|-----|------|
| **连续内存** | 所有元素紧挨着存放 |
| **固定大小** | 创建时就确定大小 |
| **随机访问** | 通过下标 O(1) 时间访问任意元素 |

### 2.3 为什么数组可以快速访问？

```
访问 arr[3] 的过程：
1. 知道数组首地址：base_address = 0x1000
2. 计算偏移量：3 × 每个元素大小（假设4字节）
3. 直接访问：0x1000 + 12 = 0x100C ✅

就像：你知道第1个柜子在哪，直接数3个柜子就到了！
```

**12岁版解释**：数组就像酒店房间，你知道第一间在哪，门牌号是连续的，所以直接数到第N间就行了。

### 2.4 数组的优缺点

```
✅ 优点：
   - 访问速度快（O(1)）
   - CPU 缓存友好（连续内存，预读效率高）

❌ 缺点：
   - 插入/删除需要移动元素（O(n)）
   - 大小固定，不能动态扩展
```

### 2.5 数组的代码实现

```python
# Python 列表就是动态数组
arr = [1, 3, 5, 7, 9]

# 访问（下标）
print(arr[2])      # 5 - O(1) 时间复杂度

# 查找元素
print(5 in arr)    # True - O(n) 时间复杂度

# 尾部插入
arr.append(11)     # O(1) 平均

# 中间插入
arr.insert(2, 4)   # 在索引2插入4 - O(n) 时间复杂度

print(arr)  # [1, 3, 4, 5, 7, 9, 11]
```

## 三、链表 —— 分散的内存空间

### 3.1 什么是链表？

**类比**：想象接力赛，每个运动员只认识自己的下一个队友。

```
链表示意图：
┌───┬───┐    ┌───┬───┐    ┌───┬───┐    ┌───┬───┐
│ 1 │ ●─┼───→│ 3 │ ●─┼───→│ 5 │ ●─┼───→│ 7 │ ✗ │
└───┴───┘    └───┴───┘    └───┴───┘    └───┴───┘
  数据 指针     数据 指针     数据 指针     数据 NULL
    ↑ head
```

**专业解释**：链表是一种线性数据结构，每个节点包含数据和指向下一个节点的指针。

### 3.2 链表的分类

```
单向链表：每个节点只指向下一个
  A → B → C → D → NULL

双向链表：每个节点指向前后两个
  NULL ← A ⇄ B ⇄ C ⇄ D → NULL

循环链表：尾节点指向头节点
  A → B → C → D → A（循环）
```

### 3.3 为什么链表插入删除快？

```
数组插入（在中间插入5）：
┌───┬───┬───┬───┬───┐
│ 1 │ 3 │ 5 │ 7 │ 9 │   ← 需要把 7,9 向后移动
└───┴───┴───┴───┴───┘

链表插入：
┌───┬───┐    ┌───┬───┐    ┌───┬───┐
│ 1 │ ●─┼───→│ 3 │ ●─┼───→│ 7 │ ●─┼───→...
└───┴───┘    └───┴───┘    └───┴───┘
                ↑
            把新节点5插进来
            只需修改两个指针！
```

**关键发现**：链表不需要挪动数据，只需修改"指针"。

### 3.4 链表的优缺点

```
✅ 优点：
   - 插入/删除只需要 O(1)（已知位置）
   - 大小可以动态增长

❌ 缺点：
   - 访问需要从头遍历（O(n)）
   - 需要额外存储指针（内存开销）
   - CPU 缓存不友好（内存分散）
```

### 3.5 链表的代码实现

```python
class Node:
    """链表节点"""
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    """单向链表"""
    def __init__(self):
        self.head = None
    
    def append(self, data):
        """尾部添加"""
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        # 找到最后一个节点
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
    
    def insert_after(self, prev_node, data):
        """在指定节点后插入"""
        new_node = Node(data)
        new_node.next = prev_node.next
        prev_node.next = new_node
    
    def delete(self, data):
        """删除第一个匹配的节点"""
        if not self.head:
            return False
        
        if self.head.data == data:
            self.head = self.head.next
            return True
        
        current = self.head
        while current.next:
            if current.next.data == data:
                current.next = current.next.next
                return True
            current = current.next
        return False
    
    def find(self, data):
        """查找节点"""
        current = self.head
        while current:
            if current.data == data:
                return current
            current = current.next
        return None
    
    def display(self):
        """显示链表"""
        elements = []
        current = self.head
        while current:
            elements.append(str(current.data))
            current = current.next
        print(" → ".join(elements) + " → NULL")
    
    def __len__(self):
        """链表长度"""
        count = 0
        current = self.head
        while current:
            count += 1
            current = current.next
        return count


# 测试
ll = LinkedList()
ll.append(1)
ll.append(3)
ll.append(5)
ll.display()  # 1 → 3 → 5 → NULL

ll.insert_after(ll.find(3), 4)
ll.display()  # 1 → 3 → 4 → 5 → NULL

ll.delete(4)
ll.display()  # 1 → 3 → 5 → NULL
```

## 四、数组 vs 链表 —— 核心对比

### 4.1 操作复杂度对比

| 操作 | 数组 | 链表 |
|------|------|------|
| 随机访问 | **O(1)** ✅ | O(n) |
| 尾部插入 | O(1) | O(n) |
| 头部插入 | O(n) | **O(1)** ✅ |
| 中间插入 | O(n) | **O(1)*** |
| 查找元素 | O(n) | O(n) |
| 删除元素 | O(n) | **O(1)*** |

*已知位置的情况下

### 4.2 内存使用对比

```
数组：
┌─────────────────────────────┐
│ 固定大小，提前分配           │
│ 如果用不完，会浪费空间       │
│ 如果不够用，无法扩展         │
└─────────────────────────────┘

链表：
┌─────┐  ┌─────┐  ┌─────┐
│数据 │→ │数据 │→ │数据 │
├─────┤  ├─────┤  ├─────┤
│指针 │  │指针 │  │NULL │
└─────┘  └─────┘  └─────┘
- 每个节点额外存储指针
- 按需分配，不浪费
```

### 4.3 场景选择

```
用数组的场景 ✅：
  - 需要频繁随机访问
  - 数据量固定
  - 对缓存命中有要求

用链表的场景 ✅：
  - 需要频繁插入/删除
  - 数据量不确定
  - 只关心头尾操作
```

### 4.4 实际例子

```python
# 场景1：频繁随机访问 → 用数组
class RandomAccessStore:
    """成绩管理系统，需要随机查看任意学生成绩"""
    def __init__(self, size):
        self.scores = [0] * size  # 固定大小数组
    
    def set_score(self, student_id, score):
        self.scores[student_id] = score  # O(1)
    
    def get_score(self, student_id):
        return self.scores[student_id]  # O(1)


# 场景2：频繁插入删除 → 用链表
class TaskManager:
    """任务管理系统，需要添加/删除任务"""
    def __init__(self):
        self.tasks = LinkedList()
    
    def add_task(self, task):
        self.tasks.append(task)  # O(n) 尾部，但可优化为 O(1)
    
    def remove_task(self, task):
        self.tasks.delete(task)  # O(1) 已知节点位置
```

## 五、经典问题

### 5.1 反转链表

```python
# 方法1：迭代法
def reverse_iterative(head):
    prev = None
    current = head
    while current:
        next_node = current.next  # 保存下一个
        current.next = prev       # 反转指针
        prev = current            # 移动 prev
        current = next_node       # 移动 current
    return prev

# 方法2：递归法
def reverse_recursive(head):
    if not head or not head.next:
        return head
    new_head = reverse_recursive(head.next)
    head.next.next = head
    head.next = None
    return new_head
```

### 5.2 链表检测环

```python
def has_cycle(head):
    """快慢指针检测环"""
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False
```

### 5.3 合并两个有序链表

```python
def merge_two_lists(l1, l2):
    """合并两个有序链表"""
    dummy = Node(0)
    current = dummy
    
    while l1 and l2:
        if l1.data <= l2.data:
            current.next = l1
            l1 = l1.next
        else:
            current.next = l2
            l2 = l2.next
        current = current.next
    
    current.next = l1 or l2
    return dummy.next
```

## 六、知识总结

### 6.1 一句话总结

```
数组 = 连续内存 + 快速访问 + 慢插入删除
链表 = 分散内存 + 慢访问   + 快速插入删除
```

### 6.2 复杂度速查

| 数据结构 | 访问 | 查找 | 插入 | 删除 |
|---------|------|------|------|------|
| 数组 | O(1) | O(n) | O(n) | O(n) |
| 链表 | O(n) | O(n) | O(1)* | O(1)* |

*已知位置

### 6.3 记忆口诀

```
数组像酒店房，连着排好找，
插入删除要挪位，固定大小不变了。

链表像接力赛，一人指下个，
插入删除只改指针，访问要从头来。
```

## 七、自我检验

### 思考题

1. **基础题**：为什么数组下标从 0 开始，而不是 1？
2. **进阶题**：如果要实现一个"头部插入效率高，但随机访问也要快"的数据结构，应该怎么设计？
3. **应用题**：LRU 缓存（最近最少使用）用什么数据结构实现最合适？为什么？
4. **思考题**：为什么 Python 的 list 是动态数组而不是链表？list 的 append 是 O(1) 吗？

### 参考答案

1. **为什么从0开始**：
   - 方便计算地址：`address = base + index × size`
   - 如果从1开始，计算时要多减1

2. **双端队列**：可以使用"数组+双向链表"的组合

3. **LRU缓存**：使用**哈希表+双向链表**
   - 哈希表提供 O(1) 查找
   - 双向链表支持 O(1) 头尾操作

4. **Python list**：
   - 因为 Python 的主要操作是随机访问和尾部操作
   - list 的 append 均摊是 O(1)，但偶尔会扩容（复制整个数组）
